Tigerstripe Explorer

The Tigerstripe Explorer provides an explorer view of all Tigerstripe projects in the current workspace. Similar to the Eclipse Package Explorer, each Tigerstripe project is represented by a tree-like structure with the following components:

  • src directory: this directory represents the root package for all artifacts in the project.
  • tigerstripe.xml: this is the Tigerstripe project descriptor. The project descriptor is a special .xml file that contains complete details about your Tigerstripe project. When you edit this file, all of the parameters for your project can be specified.
  • Tigerstripe Modules: A Tigerstripe project can have a list of dependencies set up. In this case, each module listed as a dependency will appear with a specific icon in the corresponding project hierarchy. Tigerstripe Modules are optional components.

To open the Tigerstripe Explorer

  1. Click Window, select Show View, and click Other. The Show View dialog box opens.
  2. Navigate to the Tigerstripe folder and select Tigerstripe Explorer.

Differences with Eclipse Package Explorer

Although both the Tigerstripe Explorer and the Eclipse Package Explorer may look very similar, and share a lot of the same functionalities, the following differences are worth highlighting:

  • Tigerstripe projects only: the Tigerstripe Explorer filters out any project that is not a Tigerstripe project.
  • Artifact Labels: the Tigerstripe Explorer displays all artifacts within the project packages with their corresponding icons.
  • Access to Artifact Editors: by default, when trying to open a Tigerstripe Artifact from the Tigerstripe Explorer, the corresponding specific Artifact Editor is used.
  • Publish menu: the Tigerstripe Explorer adds a publish menu option that provides access to Tigerstripe specific publish functionalities.
