Association Artifact

Description

An Association Artifact models an Association between any two artifacts in a Tigerstripe model. Each end of the Association can be further refined with the usual UML2 Association qualifiers.

Modeling Details

Standard Modeling Details

  • Package: a valid package name.
  • Name: a unique name for the Association within the scope defined by the package.
  • Super-Artifact (optional): the fully qualified name of the Association being extended by the current Association definition. This is optional.

Artifact Specific Modeling Details

  • Association Ends: each Association end supports the usual qualifiers found in UML2
    • Navigable
    • Multiplicity
    • Ordered
    • Aggregation
    • Changeable

Generated Elements

The standard OSS/J Models use References rather than Associations. As a result, standard built-in OSS/J plug-ins will ignore Assocations in a Tigerstripe Model.

Associations can be fully leveraged for generation within Custom User Plug-ins.
