Editing Exception Artifacts

After you successfully create the exception artifact, the corresponding icon appears in the Tigerstripe Explorer, and the Overview tab displays. You can further edit your newly created exception artifact in the Overview tab.

Editing an Exception Artifact

Note: Make sure that the Tigerstripe Perspective is active.

To edit an exception artifact:

  1. Double-click the exception artifact in the Tigerstripe Explorer to open the Tigerstripe Exception Artifact editor.
  2. Edit the Exception Artifact as necessary and click Save to save your changes. Alternatively, you can select File and click Save or press CTRL+S to save your changes.

You can set the following parameters in the Tigerstripe Exception Artifact editor:

  • General Information: Common to all Artifacts, the general information section includes high level information such as version, and description.
  • Attributes: This is where you can define Artifact attributes.

Exception artifacts also include a set of options to customize the generated code according to OSS/J design guidelines. For more information, refer to Exception Implementation Specifics.
