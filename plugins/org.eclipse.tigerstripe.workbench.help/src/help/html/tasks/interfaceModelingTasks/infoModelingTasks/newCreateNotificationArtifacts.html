Creating Notification Artifacts

You can create and edit Notification Artifacts through the New Notification Artifact wizard. This section details the New Notification Artifact wizard and explains how to edit your artifacts using the Tigerstripe Workbench entity perspective.

New Notification Artifact Wizard

The New Notification Artifact wizard allows you to create a new notification artifact within a Tigerstripe project. Complete the following procedure to open the New Notification Artifact wizard.

To open the New Notification Artifact wizard and create your first notification artifact:

  1. Ensure that the Tigerstripe Perspective is active. For more information, refer to Tigerstripe Perspective.
  2. Click Tigerstripe, select New, and click Notification. Alternatively, you can select the Notification icon on the Tigerstripe toolbar (). The New Notification Artifact wizard appears.
  3. Enter a name for the notification artifact in the Name text box and click Finish.

As with other types of Tigerstripe artifact wizards, the New Notification Artifact wizard requires that you enter a set of parameters before you can successfully create your notification artifact. the following outlines the information available on the New Notification Artifact wizard.

  • Source Project: the source project in which the notification artifact is being created. In most cases, Tigerstripe is able to pre-populate this field based on the context. If not populated, or if the wrong project was selected, click Browse to select the correct Tigerstripe project in which the artifact is to be created.
  • Artifact Package: each artifact is created within the scope of a package. This field must contain a valid package name. The New Notification Artifact wizard will reject all invalid package names.
  • Artifact Name: a valid name for the artifact. This name uniquely identifies the artifact within the package. The New Notification Artifact wizard will reject all invalid artifact names.
