Enumeration Constants

To edit enumeration constants, expand the Constants section in the Tigerstripe Enumeration Artifact editor. The Constants is shown below:

To add a constant:

  1. Click Add. a sample constant is created and the detailed information for the constant displays in the Enumeration Artifact editor.

To remove a constant:

  1. Select the constant from the list of defined constants.
  2. Click Remove. A confirmation dialog appears to confirm the removal of the selected constant.

To edit a constant:

  1. Select the constant from the list of defined constants.

The detailed view for the selected constant displays. You can edit the following information:

  • Constant Name: This is the name of the constant. This must be a valid identifier and cannot contain spaces or special characters. This identifier must be unique within the context of the Artifact.
  • Constant Type: The type of constant. Valid constant types are String and int. The constant types are mapped to the corresponding type for the XML and WSDL profiles. Note: In the case of an enumeration artifact, the constant type is determined by the base type of the enumeration. The base type can be set through the Details tab of the Tigerstripe Enumeration Artifact editor.
  • Constant Value: The value for the constant. If the selected type for the constant is String, the given value must be surrounded by quotes (").
  • Constant Visibility: defines the visibility of the constant.
  • Constant Description: a textual description of the constant. This field documents the purpose of the constant. You can use HTML in the field as it will be propagated into the web site where the project is to be published, as well as in the generated code. For more details on Artifact documentation, refer to Tigerstripe Model Documentation.
