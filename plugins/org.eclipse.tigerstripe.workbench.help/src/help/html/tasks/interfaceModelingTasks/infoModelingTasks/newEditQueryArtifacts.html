Editing a Named Query Artifact

After you successfully create the new named query artifact, the corresponding icon appears in the Tigerstripe Explorer, and the Overview tab displays. You can further edit your newly created artifact in the Overview tab.

Note: You can create notification attributes in the Overview tab. It is also possible to add, remove, or edit attributes on a named query artifact in the Overview tab.

Editing a Named Query Artifact

Note: Make sure that the Tigerstripe Perspective is active.

To edit an existing Named Query Artifact:

  1. Double-click the named query artifact in the Tigerstripe Explorer to open the Tigerstripe Query Artifact editor.
  2. Edit the query artifact as necessary and click Save to save your changes. Alternatively, you can select File and click Save or press CTRL+S to save your changes.

You can set the following parameters in the Tigerstripe Query Artifact editor:

  • General Information: Common to all Artifacts, the general information section covers high level information such as version, and description.
  • Attributes: This is where you can define Artifact attributes.
  • Constants: This is where you can define artifact constants.

Named Query artifacts also include a set of options to customize the generated code according to OSS/J Design Guidelines. For more information, refer to Named Query Implementation Specifics.
