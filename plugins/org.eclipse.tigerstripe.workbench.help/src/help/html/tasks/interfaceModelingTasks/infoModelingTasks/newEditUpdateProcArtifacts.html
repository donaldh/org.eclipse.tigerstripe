Editing Update Procedure Artifacts

After you successfully create the update procedure artifact, the corresponding icon appears in the Tigerstripe Explorer, and the Overview tab displays. You can further edit your newly created update procedure artifact in the Overview tab.

Note: You can create update procedure artifacts and you can also add, remove, or edit attributes on an existing update procedure artifact in the Overview tab.

Editing an Update Procedure Artifact

Note: Make sure that the Tigerstripe Perspective is active.

To edit an update procedure artifact:

  1. Double-click the update procedure artifact in the Tigerstripe Explorer to open the Tigerstripe Update Procedure Artifact editor.
  2. Edit the Update Procedure Artifact as necessary and click Save to save your changes. Alternatively, you can select File and click Save or press CTRL+S to save your changes.

You can set the following parameters in the Tigerstripe Update Procedure Artifact editor:

  • General Information: Common to all Artifacts, the general information section includes high level information such as version, description.
  • Attributes: This is where you can define Artifact attributes.
  • Constants: This is where you can define constants for your artifact.

Update Procedure artifacts also include a set of options to customize the generated code according to OSS/J design guidelines. For more information, refer to Update Procedure Implementation Specifics.
