Artifact Attributes

To edit the attributes of an artifact, expand the Attributes section in the Tigerstripe Artifact editor. The Attributes section is shown below:

Note :Not all Artifact Types support Attributes.

To add an attribute:

  1. Click Add. A blank attribute is created for the artifact attribute, and the detailed view for the attribute displays.

To remove an attribute:

  1. Select the attribute that you want to remove from the list of attributes and click Remove. A confirmation dialog appears to confirm the removal of the attribute.

To edit an attribute:

  1. Select the attribute from the list of attributes. The detailed view for the attribute displays.

The detailed attribute display contains the following information:

  • Name: the name of the attribute. This must be a valid identifier. The identifier cannot use special characters or spaces. This name must be unique within the context of this artifact.
  • Description: a textual description of the attribute. This field documents the purpose of the attribute. You can use HTML in to the field as it will be propagated into the web site where the project is to be published, as well as in the generated code. For more details on Artifact documentation, refer to Tigerstripe Model Documentation.
  • Type: the fully qualified name of the type for this attribute. To be valid, the Attribute type must be:
    • String: this denotes a String attribute. The attribute will be mapped appropriately in the XML and WSDL integration profile to ensure that there are no references to Java.
    • Primitive types: A number of Primitive types are included in the default profile, but others may be added in the profile in use for your session.
    • Other Artifact types: Some types pf artifact can be used for Attribute types - the list of valid artifacts will depend upon the profile setting for your current session, and the artifacts that already exist in your project or referenced projects.
  • Multiplicity: defines the multiplicity of an Attribute - the way in which these are used is highly generator dependent.
  • Visibility: defines the visibility of the attribute. Visibility markers signify who can access the information contained within a class.
    • Private visibility: Only classes in the same container can see and use the class. A minus sign text symbol (-) represents an attribute with Private visibility in the Class Diagram. For more information about Class Diagrams, refer to The Class Diagram Editor.
    • Public visibility: Any class that can see the container can also see and use the class. A plus sign text symbol (+) represents an attribute with Public visibility in the Class Diagram. For more information about Class Diagrams, refer to The Class Diagram Editor.
    • Protected visibility: Only classes in the same container or a descendent of the container can see and use the class. A pound sign text symbol (#) represents an attribute with Protected visibility in the Class Diagram. For more information about Class Diagrams, refer to The Class Diagram Editor.
    • Package visibility: Only classes within the same package as the container can see and use the class. A tilda sign text symbol (~) represents an attribute with Package visibility in the Class Diagram. For more information about Class Diagrams, refer to The Class Diagram Editor.

  • Qualifiers: there are three possible qualifiers that can be applied to an Attribute:
    • Readonly: an attribute marked as readonly denotes an attribute that can only be read by consumers of the Service Contract.
    • Ordered
    • Unique
  • Default Value : The default value is an optional feature that may be used by some generators. The editor does not check the value specified here against the type of the attribute.
  • Stereotypes : Stereotypes that have a scope of "Attribute" can be applied to the attribute. For more info see the section on Stereotypes.

To reorder attributes:

  1. Select the attribute from the list of attributes. Use the Up and Down buttons to place the attribute in the required position in the list of attributes. Clicking on the heading of the table will sort the list alphabetically, clicking it again will sort the list in reverse alphabetical order. Note that not all generators will necessarily use the order in a particular manner.
