Artifact Methods

To edit Artifact methods, expand the Methods section in the Tigerstripe Artifact editor. The Methods section is shown below:

Note :Not all Artifact Types support Methods.

To add a method:

  1. Click Add. A sample method is created and the detailed information for the newly created method displays in the Artifact editor.

To remove a method:

  1. Select a method from the list of defined methods.
  2. Click Remove. A confirmation dialog appears to confirm the removal of the selected method.

To edit a method:

  1. Select the method from the list of defined methods.

The detailed view for the selected constant displays. You can edit the following method information:

  • Name: the name of the method. The method name must be a valid identifier that is unique within the scope of the Artifact.
  • Description: a textual description of the method. This field documents the purpose of the method. You can use HTML code in the field as it will be propagated into the web site where the project is to be published, as well as in the generated code. For more details on Artifact documentation, refer to Tigerstripe Model Documentation.
  • Visibility: specifies the visibility of the method.
    • Private visibility: Only classes in the same container can see and use the class. A minus sign text symbol (-) represents an attribute with Private visibility in the Class Diagram. For more information about Class Diagrams, refer to The Class Diagram Editor.
    • Public visibility: Any class that can see the container can also see and use the class. A plus sign text symbol (+) represents an attribute with Public visibility in the Class Diagram. For more information about Class Diagrams, refer to The Class Diagram Editor.
    • Protected visibility: Only classes in the same container or a descendent of the container can see and use the class. A pound sign text symbol (#) represents an attribute with Protected visibility in the Class Diagram. For more information about Class Diagrams, refer to The Class Diagram Editor.
    • Package visibility: Only classes within the same package as the container can see and use the class. A tilda sign text symbol (~) represents an attribute with Package visibility in the Class Diagram. For more information about Class Diagrams, refer to The Class Diagram Editor.

  • Qualifiers: there are three possible qualifiers that can be applied to a Method:
    • Abstract
    • Ordered
    • Unique
  • Method Return
    • isVoid: If this is checked, then the method has no return, otherwise the return type is as defined below.
    • Type: If isVoid is unchecked, then the type returned by the method should be specified here. this can be either a primitive type, or another Artifact type.
    • Multiplicity: defines the multiplicity of an Attribute - the way in which these are used is highly generator dependent.
    • Return Label: in some cases a generator will need to apply a name to the return of a method.
    • Default Value : The default value is an optional feature that may be used by some generators. The editor does not check the value specified here against the return type of the method.
    • Return Stereotypes: Stereotypes that have a scope of "Argument" can be applied to the return type. For more info see the section on Stereotypes.
  • Stereotypes: Stereotypes that have a scope of "Method" can be applied to the method. For more info see the section on Stereotypes.
  • Method Arguments: to define the list of arguments for the method. The following options are available:
    • Add/Remove: use this option to add or remove arguments
    • Edit: use this option to edit the details of an argument. The fields in the dialog that pops up have the same basic use as the fields for an attribute. Stereotypes must have a scope of "Argument".
    • Up/Down: use this option to change the order of the arguments
  • Method Exceptions: to define the list of exceptions to be thrown by this method. Exceptions must be other artifact in your project. Add or remove exceptions to the list with the corresponding buttons.

To reorder methods:

  1. Select the method from the list of methods. Use the Up and Down buttons to place the method in the required position in the list of methods. Clicking on the heading of the table will sort the list alphabetically, clicking it again will sort the list in reverse alphabetical order. Note that not all generators will necessarily use the order in a particular manner.
