Publishing Service Contracts

Service Contracts are intended to be agreed to integration points part of a bigger Service Oriented Architecture. As such, their specifications needs to be made available for other to consume it, implement it and maintain or extend it. To that end, Tigerstripe Workbench allows to publish a Tigerstripe project to a web site. This web site will have the following properties:

  • Self contained web site, directly browse-able from a hard drive or ready to be uploaded to a full blown web server.
  • Direct download links to all generated Integration Profiles in a zip format.
  • Complete documentation of the Service Contract(s) contained in the published contract.

Publish to a Web Site

First, make sure the Tigerstripe Perspective is active.

Before you can publish a project to a web site, all errors related to that project must be fixed and the required publish details must be provided and enabled. Please refer to this section for more details.

Then, from the Tigerstripe Explorer, right-click on the project to publish and select Publish->Publish project...

The following wizard appears:

Simply click Finish to publish the project. For configuration and customization of the published web site, please refer to this section.
