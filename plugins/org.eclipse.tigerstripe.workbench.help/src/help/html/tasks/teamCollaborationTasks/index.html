Team Collaboration with Tigerstripe

Being part of a larger Service Oriented Architecture design, Service Contracts are usually developed by teams. Tigerstripe Workbench acknowledges that by providing mechanisms to enable collaboration during the Service Contract lifecycle.

More specifically, Tigerstripe Workbench provides support for the following:

  • Automated build environments: Tigerstripe workbench provides a Command-Line-Interface and an Apache Ant task for Tigerstripe, so Tigerstripe can be included within larger automated build environments.
  • Source Control Management integration: so that multiple architects can work on a single Service Contract design at the same time, Tigerstripe is integrated with source control management systems. An example of use with CVS is provided.
  • Release of Tigerstripe model as a web site: to facilitate documentation and access to Service Contract designs, Tigerstripe provides a mechanism to publish a Service Contract model and associated generated integration profiles onto a web site. Through this web site, other users can browse the documentation of a Service Contract without requiring access to Tigerstripe Workbench.
