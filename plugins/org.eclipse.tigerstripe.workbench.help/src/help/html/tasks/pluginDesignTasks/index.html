Plug-in Design Tasks

Tigerstripe Workbench allows you to create your own plug-ins to produce custom outputs in the same manner as built-in plug-ins. You can distribute these plug-ins to other users who then use them to generate output from thier own models. Howevver, these users cannot modify the rules encapulated in the plug-in.

Working with Plug-in Projects

The following tasks are available to your plug-in designers.

  • Creating a new Tigerstripe Plug-in Project
    • Configuring Global Properties
    • Creating and Editing rules
      • Using Filters in Rules
    • Adding a Runtime Dependency
    • Including Files in the deployed Plugin
  • Creating/Editing Velocity Templates
    • Default Velocity Context Contents
    • Using Wrappers in Templates
    • Using an Expander
  • Setting up a Plug-in log
  • Deploy and Test a Plug-in
  • Packaging a Plug-in for Distribution to Other Users
