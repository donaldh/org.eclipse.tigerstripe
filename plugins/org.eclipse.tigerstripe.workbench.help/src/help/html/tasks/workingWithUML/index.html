UML Support in Tigerstripe

Tigerstripe Workbench provides a modeling environment specially intended for Service Contract modeling. It builds on UML by defining Service Contract specific stereotypes represented by each individual Artifact type. However, Tigerstripe is not intended to be yet another UML modeling environment, but provides integration with UML, allowing to leverage any existing UML modeling environment:

  • Integration with Eclipse plugins for UML: because Tigerstripe uses an internal representation based on Java and XML, any artifact can be dragged on to a UML diagram for most existing UML Plugins into Eclipse. For more information, please contact our support group.
  • Import of existing UML Information Models into Tigerstripe projects: through XMI, Tigerstripe allows to import an existing UML Information model into a Tigerstripe project.
